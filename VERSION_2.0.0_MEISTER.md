## 🎯 WAS DER MEISTER JETZT MACHT

### Workflow eines Meisters:

**PHASE 1: Wie ein Architekt denken**
```
1. Recherche (web_search)
2. Architektur planen
3. Module definieren
4. Dependencies klären
5. Error-Szenarien vordenken
6. Testing-Strategie
→ Roadmap erstellen
```

**PHASE 2: Wie ein Meister codieren**
```
Für JEDE Datei:
1. create_file (vollständig!)
2. read_file (selbst prüfen)
3. Code Review:
   - Clean Code?
   - Best Practices?
   - Performance OK?
   - Security OK?
4. modify_file wenn nicht perfekt
5. update_roadmap_status
```

**PHASE 3: Wie ein Profi testen**
```
1. Syntax Check (alle Dateien)
2. Logic Validation (Dateien lesen)
3. Integration Test
4. Preview Test (visuell)
5. UX Test (wie echter User)
6. Gameplay Test (bei Spielen)
→ WENN FEHLER: Debug + Fix + RE-TEST!
```

**PHASE 4: Qualitätskontrolle**
```
1. Alle Dateien nochmal lesen
2. Code Review (Clean Code?)
3. Architecture Review (macht Sinn?)
4. Performance Check
5. Security Check
6. User Experience Check
→ NUR WENN ALLES ✅: mark_complete
```
